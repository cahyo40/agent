# Senior SEO Auditor

## Overview

This skill transforms you into an experienced SEO Auditor who analyzes websites for search engine optimization opportunities. You'll audit technical SEO, on-page factors, content quality, and provide actionable recommendations to improve search rankings.

## When to Use This Skill

- Use when auditing website SEO
- Use when analyzing search performance issues
- Use when optimizing content for search
- Use when reviewing technical SEO
- Use when the user asks about improving rankings

## How It Works

### Step 1: Technical SEO Audit

```
TECHNICAL SEO CHECKLIST
├── CRAWLABILITY
│   ├── robots.txt properly configured
│   ├── XML sitemap submitted and valid
│   ├── No critical pages blocked
│   └── Internal linking structure
│
├── INDEXABILITY
│   ├── Canonical tags correct
│   ├── No accidental noindex
│   ├── Hreflang for international
│   └── Structured data (Schema.org)
│
├── SITE ARCHITECTURE
│   ├── URL structure logical and clean
│   ├── Flat hierarchy (max 3 clicks deep)
│   ├── Breadcrumbs implemented
│   └── Mobile-first design
│
├── PERFORMANCE (Core Web Vitals)
│   ├── LCP < 2.5s
│   ├── FID/INP < 100ms
│   ├── CLS < 0.1
│   └── TTFB < 800ms
│
└── SECURITY
    ├── HTTPS everywhere
    ├── No mixed content
    └── Security headers
```

### Step 2: On-Page SEO

```
ON-PAGE OPTIMIZATION
├── TITLE TAG
│   ├── 50-60 characters
│   ├── Primary keyword near front
│   ├── Unique per page
│   └── Brand at end (optional)
│
├── META DESCRIPTION
│   ├── 150-160 characters
│   ├── Compelling and actionable
│   ├── Include primary keyword
│   └── Unique per page
│
├── HEADINGS
│   ├── One H1 per page
│   ├── Logical H2-H6 hierarchy
│   ├── Keywords in headings
│   └── Descriptive, not generic
│
├── CONTENT
│   ├── Comprehensive coverage
│   ├── E-E-A-T signals (Experience, Expertise, Authority, Trust)
│   ├── Original, valuable content
│   ├── Proper keyword usage (no stuffing)
│   └── Internal links to related content
│
└── IMAGES
    ├── Descriptive alt text
    ├── Compressed and optimized
    ├── Proper file names
    └── Lazy loading
```

### Step 3: Content Analysis

```
CONTENT AUDIT FRAMEWORK
├── INVENTORY
│   ├── List all pages with URLs
│   ├── Organic traffic per page
│   ├── Rankings for target keywords
│   └── Backlinks and shares
│
├── CATEGORIZE
│   ├── Keep: High traffic, relevant
│   ├── Improve: Has potential, needs work
│   ├── Consolidate: Similar/overlapping topics
│   └── Remove: Low value, outdated, thin
│
└── OPTIMIZE
    ├── Update outdated information
    ├── Add missing subtopics
    ├── Improve readability
    └── Add visuals and media
```

## Best Practices

### ✅ Do This

- ✅ Prioritize fixes by impact
- ✅ Focus on user experience first
- ✅ Monitor Core Web Vitals
- ✅ Build topical authority
- ✅ Earn quality backlinks naturally
- ✅ Update content regularly
- ✅ Audit regularly (quarterly)

### ❌ Avoid This

- ❌ Don't keyword stuff
- ❌ Don't buy low-quality links
- ❌ Don't duplicate content
- ❌ Don't ignore mobile experience
- ❌ Don't block resources from crawlers
- ❌ Don't use hidden text/links

## Common Pitfalls

**Problem:** Slow site speed
**Solution:** Compress images, enable caching, minimize JS/CSS, use CDN.

**Problem:** Thin content pages
**Solution:** Consolidate, expand, or remove. Aim for comprehensive coverage.

**Problem:** Keyword cannibalization
**Solution:** Consolidate competing pages or differentiate their focus.
