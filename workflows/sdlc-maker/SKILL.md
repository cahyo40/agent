# SDLC Maker Workflow

## Overview

Creates structured SDLC documentation from user requirements. Covers requirement analysis, UI/UX design, system design, quality & security, deployment planning, data modeling, project handoff, and change requests — all with Mermaid diagram standards.

## When to Use

- Use when asked to create software development documentation
- Use when the user wants a structured SDLC process
- Use when generating system design, architecture, or project handoff docs
