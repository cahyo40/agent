# Workflow: Test Execution & Validation

## Agent Behavior

When executing this workflow, the agent MUST:
- Jalankan test suite dengan command yang sesuai tech stack
- Parse output untuk extract failures, errors, dan coverage
- Untuk setiap failure, diagnosa root cause
- Generate fix untuk test ATAU kode (mana yang salah)
- Iterate sampai semua tests pass
- Update coverage tracking

## Overview

Comprehensive test execution workflow. Menjalankan test suite, menganalisis hasil, dan membantu fix test yang gagal. Workflow ini mendukung **TDD cycle** yang sudah diwajibkan di pipeline.

## Input

- **Project directory** — Root folder project
- **Checklist file (optional)** — CHECKLIST.md untuk update status
- **RULE.md** — Untuk validasi pattern jika test gagal karena kode salah

## Output

- `{output_dir}/TEST_REPORT.md` — Hasil test execution + coverage
- Updated CHECKLIST.md (jika ada)
- Fixed test/kode files

## Steps

### Step 1: Deteksi Test Framework

Deteksi tech stack dan test framework dari project:

| Stack | Test Command |
|-------|-------------|
| Flutter/Dart | `flutter test` |
| Next.js/React | `npm test` or `npx jest` |
| Vue/Nuxt | `npm run test` |
| Go | `go test ./...` |
| Python | `pytest` |
| Laravel | `php artisan test` |

### Step 2: Jalankan Test Suite

```bash
{test_command} 2>&1
```

Capture full output.

### Step 3: Parse Results

Parse output untuk extract:

```markdown
## 📊 Test Results

| Metric | Value |
|--------|-------|
| Total Tests | XX |
| Passed | XX |
| Failed | XX |
| Errors | XX |
| Skipped | XX |
| Coverage | XX% |
| Duration | XXs |
| **Status** | **✅ ALL PASSED / ❌ FAILURES DETECTED** |
```

### Step 4: Diagnosa Setiap Failure

Untuk setiap test failure:

```markdown
### Failure {#}: {Test Name}
**File:** `path/to/test_file.dart:{line}`
**Expected:** {what the test expected}
**Actual:** {what actually happened}
**Root Cause:** {why it failed — kode salah / test salah / missing feature}

**Diagnosis:**
- [ ] Test assertion correct?
- [ ] Implementation matches expectation?
- [ ] Missing widget/state in tree?
- [ ] Async timing issue?
- [ ] Mock not set up correctly?

**Fix Action:**
{minimal fix description}
```

### Step 5: Generate Fix

Berdasarkan diagnosis:

| Root Cause | Fix Action | Example |
|------------|-----------|---------|
| **Kode salah** | Fix implementasi | `return value` instead of `return null` |
| **Test salah** | Fix assertion | `expect(result, equals(expected))` |
| **Missing widget** | Add widget ke tree | Wrap with `MaterialApp` |
| **Async timing** | Add `pumpAndSettle` | `await tester.pumpAndSettle()` |
| **Mock issue** | Fix mock setup | `when(mock.method()).thenReturn(value)` |

### Step 6: Iterate

1. Apply fix
2. Re-run specific test: `{test_command} --name "{test_name}"`
3. If pass → next failure
4. If still fail → re-diagnose
5. After all individual fixes → run full suite

### Step 7: Generate Test Report

```markdown
# 🧪 Test Execution Report
# {Nama Project}

> **Generated by vibe-coding-toolkit/11_test_execution.md**

---

## 📊 Final Results

| Metric | Value |
|--------|-------|
| Tests | XX / XX passed |
| Coverage | XX% |
| Duration | XXs |
| **Status** | **✅ ALL PASSED** |

---

## 🔧 Fixes Applied

| Test | Issue | Fix |
|------|-------|-----|
| test_name | Wrong assertion | Updated expect() call |

---

## 📈 Coverage by Layer

| Layer | Coverage | Status |
|-------|----------|--------|
| Unit (Data) | XX% | ✅ Met target (80%) |
| Unit (Domain) | XX% | ✅ Met target (80%) |
| Widget | XX% | ⚠️ Below target (70%) |
| Integration | XX% | ❌ Below target (50%) |

---

## ✅ Next Steps
- [ ] Improve widget test coverage
- [ ] Add integration tests for critical flows
```

### Step 8: Update Checklist

Jika CHECKLIST.md tersedia:
- Tandai task testing yang sudah selesai
- Update coverage metrics

## Quality Criteria

- Setiap failure HARUS didiagnosa root cause-nya — jangan asal re-run
- Fix minimal — jangan refactor seluruh file
- Verifikasi dengan re-run test
- Full suite wajib pass sebelum claim selesai
